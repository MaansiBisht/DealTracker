your behalf вАФ every 30 seconds for products, every 3 hours for hotels вАФ

## рЯЪА Quickstart

### рЯР≥ Docker вАФ recommended

The whole stack is one image: Node builds the UI, Python serves the
API, headless Chromium does the scraping.

```bash
git clone https://github.com/MaansiBisht/DealTracker.git
cd DealTracker
cp .env.example .env       # fill in real values, see Configuration вЖУ
docker compose up -d --build
docker compose logs -f app
```

Open `http://<host>:8000`. Behind nginx/Caddy, reverse-proxy `/` and
`/api/*` to port 8000 and you've got TLS + a domain.

Update later:
```bash
git pull && docker compose up -d --build
```

### вЪ° Local dev

Two processes вАФ `uvicorn` (API on :8000) and `vite` (HMR UI on :5173).

```bash
# 1. Python backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn src.server.main:app --reload --port 8000

# 2. React frontend (separate terminal)
cd ui && npm ci && npm run dev
```

Open `http://localhost:5173`. Vite proxies `/api` and `/events` to uvicorn.

### рЯЦ•пЄП CLI

Interactive prompt-driven flow вАФ same scrapers as the web app, no
server, no scheduler. Good for one-off checks from a laptop.

```bash
pip install -r requirements.txt
cp .env.example .env
python main.py        # interactive prompts
```

## рЯОЫпЄП Configuration

Every knob is an env var. Real defaults live in `.env.example`.

| Variable                   | Required | What it does                                              |
| -------------------------- | :------: | --------------------------------------------------------- |
| `EMAIL_ADDRESS`            |    вЬЕ    | Gmail account that sends the alerts                       |
| `EMAIL_PASSWORD`           |    вЬЕ    | **Gmail App Password** ([generate here][app-pw])           |
| `PINCODE`                  |    вЬЕ    | Used by Amul scraper before reading stock/price           |
| `WEB_USER` ¬Ј `WEB_PASS`    |   prod   | HTTP Basic auth for the ops console                       |
| `FALLBACK_EMAIL`           |    вП§    | Retried when delivery to the watch's email fails          |
| `TICK_INTERVAL_PRODUCT_SEC`|    вП§    | Default `30` (dev). Use `3600` in prod.                   |
| `TICK_INTERVAL_HOTEL_SEC`  |    вП§    | Default `60` (dev). Use `10800` in prod.                  |
| `DATABASE_URL`             |    вП§    | Default `sqlite:////app/data/dealtracker.db`              |
| `LOG_LEVEL`                |    вП§    | `DEBUG` / `INFO` / `WARNING` for the operator log         |
| `CHROME_BIN`               |    вП§    | Chromium path. The Docker image sets this for you.        |

> рЯФР **Why an App Password?** Google blocks plain SMTP login from your

## рЯЫ†пЄП Add a new scraper

It's three steps. The codebase already has 9 examples to copy from.

<details>
<summary><b>рЯУ¶ Product scraper</b></summary>

1. Create `src/scrapers/newsite.py`:
   ```python
   def scrape_newsite(driver, url):
       driver.get(url)
       # ...your parsing logic...
       return {
           "title": "Optional product name",
           "price": "1234.56",
           "stock_status": "in stock",   # or "out of stock"
       }
   ```

2. Wire it in `src/scrapers/__init__.py`:
   ```python
   from .newsite import scrape_newsite
   SCRAPERS = {..., "newsite": scrape_newsite}
   PLATFORM_PATTERNS = {..., "newsite.com": "newsite"}
   ```

3. Lock the routing in `tests/test_routing.py` so a typo doesn't unship it later.

</details>

<details>
<summary><b>рЯП® Hotel scraper</b></summary>

Same shape, plus append `"newhotel"` to `HOTEL_PLATFORMS`. The runner
will pick `scan_hotel_prices_monthly` instead of single-shot scraping.
Returned dict adds `"type": "hotel"` and may include `"rating"`.

</details>
